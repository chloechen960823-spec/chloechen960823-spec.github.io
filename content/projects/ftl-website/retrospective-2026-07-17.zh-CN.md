# FTL Group 官网作品集页 - 2026-07-17 设计复盘

> 记录类型：过程复盘，不是公开 Case Study 文案  
> 对应网站版本：`folded-v6`  
> 记录目的：保留今天有效的判断、失败的尝试和下一轮验收标准，避免重复消耗。

## 1. 今天完成了什么

- 将 FTL Group 官网作为独立项目加入 Work 页面；
- 从旧 XD、Figma、FigJam、PPT/PDF 和历史作品集素材中整理了桌面端、移动端、旧站、站点树、wireframe、UI kit 等资产；
- 建立了 FTL 项目的中文事实源，区分已证实事实、口述信息、待确认内容和不可公开的推断；
- 项目页已经覆盖业务背景、旧站问题、信息架构、设计判断、响应式成果、Webflow/CMS 交付、上线事实和证据边界；
- 尝试了设备样机、长页面自动滚动、折叠面板、滚动进入和响应式页面联动展示。

## 2. 保留下来的优点

### 视觉语言没有被参考网站带跑

项目页继续使用作品集既有的字体、颜色、网格、留白和导航系统。参考网站主要用于研究项目结构与交互方式，没有直接复制对方的视觉风格。这一点需要继续保持。

### 内容开始从“流程清单”转向“设计判断”

当前叙事不再只说做了 persona、wireframe 和 UI，而是尝试解释：

- 为什么官网需要从内部服务目录转向客户的业务情境；
- 为什么先重构信息架构，再进入视觉设计；
- 为什么首页应该承担分流，而不是解释全部业务；
- 为什么模块化交付和 CMS 对小团队与后续运营有价值；
- 哪些结果能够证明，哪些增长结果不能虚构。

### 素材使用范围比早期版本更完整

页面不再只展示一张首页截图，已经纳入旧站、站点树、wireframe、UI system、桌面和移动页面，也保留了继续补充 benchmark、persona 与 UX 过程证据的空间。

## 3. 今天最挫败的部分

### 动效仍然像“滚动一个 JPG”

这是今天最准确、也最重要的评价。页面虽然加入了自动滚动、位移、折叠和淡入，但素材本身的状态没有发生有意义的变化。技术上有动画，体验上仍然接近 PPT 或长图浏览。

根本原因不是动画数量不足，而是缺少：

- 连续的场景转换；
- 前景、中景、背景之间的空间层级；
- 与叙事同步的滚动编排；
- 进入、停留、转折、离开的节奏；
- 从一种产品状态自然演化到下一种状态的视觉逻辑。

### 部分形式大于内容

- 深绿色大区块占据过多空间，但只承载少量文字；
- 早期的 1/2/3/4 交互和 tab 需要用户主动点击，打断了连续阅读；
- SWOT 四宫格、结构图与静态截图仍然偏平铺，像在陈列材料而不是讲述判断；
- 旧站样机与工作区截图的组合缺少明确的视觉关系，既没有形成对比，也没有形成拆解过程；
- 某些设备比例、底座和屏幕尺寸显得粗糙，削弱了原始设计本身的可信度；
- 长截图自动滚动的速度与停止位置不稳定，容易卡在没有信息价值的区域。

### 参考被理解成“添加效果”，没有被转化成 motion system

今天较多时间花在局部加效果，却没有先定义整页的运动语言。结果是每一段各自动，但彼此没有连续关系。这也是页面看起来没有真正成为 Interactive Exhibition 的原因。

## 4. 当前版本为什么仍然值得保存

`folded-v6` 不是最终答案，但它已经形成一个可访问、可继续迭代的项目页，并保存了今天整理出的内容结构和真实素材。现在上线它，比继续疲劳修改更有价值：

- 它让作品集项目数量和完整度向 MVP 前进；
- 它保留了当前最可信的项目事实与证据边界；
- 它提供了一个可以对照改进的稳定基线；
- 它避免明天重新从混乱素材开始。

## 5. 下一轮 motion 方向

参考：

- `https://heeyeunsongdesigner.framer.website/`
- `https://www.heystephen.me/`

下一版不以“再加几个动画”为目标，而是建立五个连续场景：

1. **Hero / 展品出现**：缩小设备尺度，通过透视、轻微悬浮和滚动展开，让设备从展品自然变成完整网站；
2. **Before / 问题暴露**：旧站通过折叠、拆解或遮罩逐步暴露信息问题，而不是静态摆放一张截图；
3. **Evidence / 证据编排**：benchmark、persona、SWOT 与站点树在同一个固定舞台中随滚动依次替换；
4. **Transformation / 原位演化**：Structure → Wireframe → UI 在同一构图中连续变化，不再使用三个独立面板；
5. **Responsive / 形态重组**：桌面界面在滚动中压缩并重组为移动端，同时保持内容层级对应。

## 6. 下一轮验收标准

- 不需要点击也能顺畅理解主要故事；
- 每一次运动都解释一个内容变化，而不是单纯装饰；
- 截图不能只是从上滚到下，必须发生裁切、聚焦、拆解、对比或形态转换；
- 同一屏只保留一个主要视觉焦点；
- 动效开始前、进行中和结束后都不能出现空白、重影或无意义停顿；
- 桌面与移动端画面尺寸真实、清晰，不靠过度放大撑场面；
- 使用 transform、opacity、mask 和少量 sticky 场景控制性能，不为炫技引入高负荷效果；
- 支持 `prefers-reduced-motion`，动效关闭后内容仍然完整可读；
- 在桌面与移动视口进行真实滚动验收后才能提交。

## 7. 明天继续时不需要重新讨论的结论

- 当前版本先上线，不继续疲劳修改；
- 视觉系统继续沿用作品集现有语言；
- 项目内容必须以真实事实、设计判断和证据为核心；
- FTL 页面下一轮的主要问题是 motion choreography，不是配色或再增加素材；
- 用户后续提供 3–5 个喜欢的具体动效片段即可，不需要先寻找或安装新的 motion skill；
- 未回答的项目问题继续保留在知识库中，集中答疑，不在每次制作中反复追问。
